     140             :  * Guide for the inline assembly below:
     141             :  *
     142             :  * x86 instructions are in the form "{opcode}{length} {source}, {destination}",
     143             :  * where the length is one of the letters "b" (byte), "w" (word, 16-bit), "l"
     144             :  * (dword, 32-bit), "q" (qword, 64-bit).
     145             :  *
     146             :  * In most cases, we can omit the length because it's inferred from one of the
     147             :  * registers. For example, "xchg %0,%1" doesn't need the length suffix because
     148             :  * we can only exchange data of the same size and one of the operands must be a
     149             :  * register.
     150             :  *
     151             :  * The exception is the increment and decrement functions, where we add and
     152             :  * subtract an immediate value (1). For those, we need to specify the length.
     153             :  * GCC and ICC support the syntax "add%z0 $1, %0", where "%z0" expands to the
